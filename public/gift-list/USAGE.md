# 使用指南

## 快速开始

### 方式一：直接访问（开发环境）
如果你在本地开发环境中，可以直接在浏览器中打开：
```
/home/engine/project/public/gift-list/index.html
```

### 方式二：通过 Web 服务器访问（推荐）
启动项目的开发服务器后，访问：
```
http://localhost:端口号/gift-list/
```

## 使用场景

### 场景 1：作为创建者

**步骤 1：创建清单**
1. 打开应用首页
2. 填写以下信息：
   - 清单标题：例如"张三的30岁生日礼物清单"
   - 寿星姓名：例如"张三"
   - 生日日期：选择生日日期
3. 点击"创建清单"按钮

**步骤 2：添加礼物**
1. 创建成功后自动跳转到清单页面
2. 在"添加礼物"表单中填写：
   - 礼物名称：例如"AirPods Pro"
   - 价格：例如"1999"
   - 购买链接：例如"https://example.com/airpods"
   - 备注：例如"白色版本"
3. 点击"添加礼物"按钮
4. 重复以上步骤添加更多礼物

**步骤 3：分享清单**
1. 在清单顶部的"分享链接"卡片中
2. 点击"复制"按钮复制链接
3. 将链接通过微信、邮件等方式发送给朋友

**步骤 4：管理礼物（可选）**
- 点击礼物卡片上的 ✏️ 按钮编辑礼物信息
- 点击礼物卡片上的 🗑️ 按钮删除礼物
- 随时添加新的礼物

### 场景 2：作为访客（认领者）

**步骤 1：访问清单**
1. 点击朋友分享的链接
2. 浏览器会自动打开清单页面
3. 查看寿星信息和礼物列表

**步骤 2：认领礼物**
1. 浏览礼物列表
2. 找到你想认领的礼物
3. 点击"点击认领"按钮
4. 在弹出的对话框中输入你的姓名
5. 点击"确认认领"按钮

**步骤 3：确认认领状态**
- 认领成功后，礼物卡片会显示"✓ 已被 你的名字 认领"
- 已认领的礼物其他人无法再次认领
- 礼物卡片背景会变成绿色，表示已被认领

## 常见问题

### Q1: 我换了设备，为什么无法编辑清单了？
**A:** 创建者身份是通过浏览器的 localStorage 存储的，更换设备或浏览器后会失去创建者权限。如果需要在多设备上管理，请使用同一个浏览器，或者在原设备上操作。

### Q2: 如果有人恶意认领所有礼物怎么办？
**A:** 目前应用没有实现撤销认领功能。创建者可以删除被恶意认领的礼物，然后重新添加。未来版本可能会增加创建者取消认领的功能。

### Q3: 清单会保存多久？
**A:** 清单数据存储在浏览器的 localStorage 中，除非手动清除浏览器数据或卸载浏览器，否则会永久保存。

### Q4: 可以创建多个清单吗？
**A:** 可以！每次创建新清单时，系统会生成一个唯一的 ID。你可以为不同的朋友创建不同的清单。

### Q5: 如何找回之前创建的清单？
**A:** 在首页点击"查看现有清单"链接，输入清单 ID（从分享链接中获取，格式为 `?list=xxxxx` 中的 `xxxxx` 部分）即可访问。

### Q6: 礼物的价格必须填写吗？
**A:** 不需要！所有礼物信息（名称、价格、链接、备注）都是可选的。但建议至少填写礼物名称，以便访客了解礼物内容。

### Q7: 可以取消已认领的礼物吗？
**A:** 访客无法取消认领。如果需要取消，请联系清单创建者，由创建者删除该礼物并重新添加。

### Q8: 分享链接会过期吗？
**A:** 不会过期。只要清单数据还在浏览器的 localStorage 中，链接就始终有效。

### Q9: 可以设置清单密码吗？
**A:** 当前版本不支持密码保护。任何知道链接的人都可以查看和认领礼物。请注意保护链接隐私。

### Q10: 支持哪些浏览器？
**A:** 支持所有现代浏览器，包括：
- Chrome/Edge 90+
- Firefox 88+
- Safari 14+
- iOS Safari
- Chrome Mobile

## 技术说明

### 数据存储位置
- 浏览器 LocalStorage
- 键名格式：`gift_list_{hashId}`
- 创建者 Token：`creator_token`

### 清单 ID 格式
- 长度：8 位
- 字符集：数字和小写字母（36进制）
- 生成方式：基于时间戳和清单信息的哈希算法

### 数据安全
- 所有数据仅存储在本地浏览器中
- 无服务器端存储
- 无用户账号系统
- 建议不要在清单中填写敏感信息

## 最佳实践

1. **清单标题要明确**：使用易于识别的标题，如"张三的生日礼物清单"
2. **礼物信息要详细**：尽可能填写完整的礼物信息，包括链接和备注
3. **及时分享链接**：创建清单后尽快分享，避免遗忘
4. **定期检查认领状态**：创建者可以定期查看哪些礼物已被认领
5. **保存清单链接**：创建者建议保存清单链接，方便后续访问
6. **提醒朋友填写姓名**：认领时请朋友使用真实姓名，方便识别
